Workflow

One encounter in. A usable clinical draft out.

The product is intentionally narrow: capture the conversation, shape the draft, preserve clinician control, and make export into existing journal workflows straightforward.

01

Record once

Start with a single, native recorder surface on iPhone. No cluttered workspace, no setup wall before the visit, no need to choose between cloud and local until the recording is done.

02

Draft in the right lane

Stay on-device with WhisperKit and MLX Qwen, or use the managed cloud path on Eir servers in Sweden with Berget AI for transcription and note inference.

03

Review before sign-off

Edit the note by hand, inspect previous versions, reopen earlier encounters, and export journal-ready text or FHIR-compatible payloads without pretending the draft is final.

Privacy architecture

Built for healthcare constraints, not demo theater.

Cloud mode

Audio is routed through Eir-managed infrastructure hosted in Sweden. Eir is configured for zero Eir-side data retention, while Berget AI performs the inference work for transcription and note generation.

  • Fly deployment in Stockholm
  • Bearer-token gated API surface
  • Clear separation between Eir control plane and inference provider

Local mode

The same app can download models onto the iPhone and run transcription plus drafting locally. That gives teams a credible offline path without forking the product into a separate build.

  • WhisperKit for transcription
  • MLX Qwen for local drafting
  • Thoughtful progress states for model preparation

Outputs

Swedish-first, but not Swedish-only.

The current product is optimized around Swedish documentation patterns, accent-insensitive transcript search, and journal-style note structures. The language pipeline remains multilingual by design.

Journal-ready text

Fast paste into systems like Cosmic, TakeCare, Melior, or whatever the clinic already runs.

FHIR export

Structured DocumentReference export for teams building deeper integrations over time.

History and versions

Saved encounters, editable drafts, revision timelines, and visible recovery when a note changes.

Open deployment model

Run it yourself, bring your own keys, or use an Eir-managed Sweden-hosted deployment path.

Deployment

Open source, but ready to be bought for real use.

Some teams want to self-host. Some want a clinic-ready hosted path with Sweden-first infrastructure, controlled spend, and a cleaner support model. Eir Scribe is designed to support both without splitting the product in two.
