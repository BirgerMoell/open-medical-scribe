Eir Scribe Support

For the open-source build of Eir Scribe, support requests and bug reports should go through the project repository.

  • General bugs and usability issues: open a GitHub issue.
  • Privacy or deployment questions: review the privacy policy, then open an issue with the relevant context.
  • Clinical deployment support: contact the healthcare organization or operator that configured your backend, retention policy, and journal import workflow.

If this app is distributed by a healthcare provider or company, that operator should replace or extend this page with their own support contacts, response expectations, and incident process.
