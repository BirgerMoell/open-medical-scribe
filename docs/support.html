Support

Open source at the core, with a hosted path clinics can actually run.

For the open-source build, support starts in the repository. For production or clinical deployments, the operator of the backend should own response times, incident handling, retention policy, and journal import workflows.

Product issues

Bug reports, UX issues, and reproducible failures in the open-source codebase should go through the project repository.

Privacy questions

Review the privacy policy first, then open an issue with the deployment context, processing mode, and backend setup that matters.

Clinical deployment

If Eir Scribe is distributed by a healthcare organization or company, that operator should publish its own support contacts, deletion handling, and incident process.
