# Complaint Handling Procedure

Document ID: OMS-MDR-CL1-COMPLAINT-HANDLING-PROCEDURE
Version: 0.1
Status: Draft
Owner: Birger Moëll

Last updated: 2026-03-11

This procedure defines how complaints about the OMS Class I Web Reference Build are received, assessed, investigated, and closed.

## Purpose

To ensure that complaints are handled consistently and that safety-relevant information is fed into PMS, vigilance, CAPA, and risk management.

## Intake channels

- support email
- support portal or ticketing system
- direct reports from customers or partner healthcare providers
- internal escalation from monitoring or operations

## Minimum complaint record fields

- date received
- reporter identity and organization if available
- affected release/version
- affected workflow step
- summary of issue
- safety relevance assessment
- incident/vigilance assessment
- investigation outcome
- closure date

## Triage categories

- non-safety defect
- safety-relevant complaint
- possible serious incident
- security/privacy event needing parallel handling

## Investigation expectations

Investigation should determine:

- whether the issue is reproducible
- whether it relates to validated behavior or production drift
- whether a risk control failed
- whether labeling, training, or design changes are needed

## Closure requirements

A complaint may be closed only when:

- investigation is documented
- required escalations are made
- corrective action decision is documented
- PMS / risk file updates are triggered where needed
